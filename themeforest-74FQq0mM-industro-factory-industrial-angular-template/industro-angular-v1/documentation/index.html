Introduction

Industro Angular is the Industry base most powerful and responsive template having great modules and very useful functions with an attractive design. This theme provides powerful flexibility to customize code as per your requirement because of our code simplicity.

We provide five very impressive industry home pages with additional 15+ other module pages related to industry life cycle.

This document will describe whole Industro template with an application structure and guide how to use all the modules. This will beneficial for anyone who can update the template as per own unique need.

We have used very impressive and vast technology named “Angular”, It provides extremely high speed performance through single page application.

Angular Industro can be customized at any level, it includes custom sections and reusable elements for every single page. A new component or element or section can be added using angular's component generator and then, can be customized/modified as per requirement using reusable elements. For reusability template uses bootstrap and custom reusable styles.

The entire template uses .html and typescript (.ts) files for customization. Further for page functionalities like sliders, scrolls etc. javascript (.js) files are used and for designs and responsive patterns .css files are used.

The theme/color switcher shown in preview template will not be included in download package, as it is only for preview purpose.

File & Folder Structure

This describes the file structure of Industro Angular application template.

src The root directory of the application, contains source and necessary assets used for application.
—— app Entire source code of the application is available in this folder so it is the main folder.
———— directives This directory contains custom actions/functions used to perform different tasks in different components.
———— elements All pages are divided into small sections, and they all are constructed using custom angular components. This directory contains such components. Any new component can be added here and used in different pages.
———— handlers This directory contains custom actions/functions handlers i.e. mail, forms etc.
———— modules This directory contains moduler routing for similar groups/sections.
———— pages All pages of this application contained in this directory.
———— sections This directory contains fractions of enitre template, to make arrangement and mantinence in better way.
———— app-routing.module A file named app.routing.module defines all routes of the application with respect to related components or pages.
———— app.component.html The root component of the application. Initial routing is defined here.
———— app.module This directory contains declaration of all components/pages for the application. A component need to be defined here before using in application.
—— assets This contains necessary Assets used for application.
———— css This contains CSS. CSS stands for Cascading Style Sheets. It’s used to define styles for web pages including the design, layout, and variations in display for different devices and screen sizes.
———— fonts This contains fonts including font awesome library used for the application.
———— images This contains images used for the application.
———— js This contains javascript files used for the application.
———— plugins This contains plugins used for the application.
—— index.html This contains necessary Assets used for application.
angular.json A file named angular.json at the root level of an Angular workspace provides workspace-wide and project-specific configuration defaults for build and development tools provided by the Angular CLI. Path values given in the configuration are relative to the root workspace folder.
package.json A package.json file contains metadata about an app or module. It includes the list of dependencies so it’s most important for the application.

 

Installation

This section guides you to install all required software and libraries to utilize Industro angular for your projects fully.

Node

Node.js® is a JavaScript run time built on Chrome’s V8 JavaScript engine. Node.js is required to build the Industro Angular. Download the latest version of Node and install it: Download Nodejs

NPM

NPM is the package manager for JavaScript and the world’s largest software registry. NPM is a separate project from Node.js and tends to update more frequently. As a result, even if you’ve just downloaded Node.js (and therefore npm), you’ll probably need to update your NPM .

npm install --global npm@latest

Verify that npm in successfully installed, and version of installed npm will appear

npm --version

Angular

Installation

1. First, Ensure to install the Angular CLI, open a terminal window and run the following command:

npm install -g @angular/cli

2. Then after, unzip the zip file that you have download from ThemeForest. You will find a Industro-Angular-1.0 project folder.

3. Extract zip File (Industro.zip) into a folder, you have to work within.

4. Open a Node Prompt. Go to your Working Folder, And run the Following Command and wait for finish installation.

npm install

5. This command will install all the required node module into node_module/ inside your working folder.

6. You are ready to run the Industro angular application for the first time

7. While still your working folder and run the following command

ng serve --open
Deployment

For the simplest deployment, create a production build and copy the output directory to a web server.

1. Start with the production build:

ng build --prod
    or
ng build --prod --base-href "<your-domain>"

2. Copy everything within the output folder (dist/project-name/ by default) to a folder on the server.

3. Configure the server to redirect requests for missing files to index.html. Learn more about server-side redirects at here.

This is the simplest production-ready deployment of your application. Further help

To get more help : https://angular.io/start

Adding New Component

The easiest way to create a component is with the Angular CLI. You can also create a component manually. Full overview about angular components can be find here.

Creating a component using the Angular CLI

To create a component using the Angular CLI:

Step 1 : From a terminal/cmd window, navigate to the directory containing your application.

Step 2: Run the ng generate component <component-name> command, where <component-name> is the name of your new component.

By default, this command creates the following:

  • A folder named after the component
  • A component file, <component-name>.component.ts
  • A template file, <component-name>.component.html
  • A CSS file, <component-name>.component.css
  • A testing specification file, <component-name>.component.spec.ts
Where <component-name> is the name of your component.

Creating a component manually

Although the Angular CLI is the easiest way to create an Angular component, you can also create a component manually.

To create a new component manually:

Step 1 : Navigate to your Angular project directory.

Step 2 : Create a new file, <component-name>.component.ts.

Step 3 : At the top of the file, add the following import statement. 

                    import { Component } from '@angular/core';

Step 4 : After the import statement, add a @Component decorator. 

                    @Component({
                    })

Step 5 : Choose a CSS selector for the component. 

                    @Component({
                        selector: 'app-component-overview',
                      })

For more information on choosing a selector, see Specifying a component's selector.

Step 6 : Define the HTML template that the component uses to display information. In most cases, this template is a separate HTML file. 

                    @Component({
                        selector: 'app-component-overview',
                        templateUrl: './component-overview.component.html',
                      })

For more information on defining a component's template, see Defining a component's template.

Step 7 : Select the styles for the component's template. In most cases, you define the styles for your component's template in a separate file. 

                    @Component({
                        selector: 'app-component-overview',
                        templateUrl: './component-overview.component.html',
                        styleUrls: ['./component-overview.component.css']
                      })

Step 8 : Add a class statement that includes the code for the component. 

                    export class ComponentOverviewComponent {

                    }
